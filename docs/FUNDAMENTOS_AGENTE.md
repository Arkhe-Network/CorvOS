## 🌿 GIT: O REGISTRO DA EVOLUÇÃO

Versionamento é a gestão do tempo e das linhas temporais (branches) do teu projeto.

### 💾 Opcodes de Evolução (Comandos Git)

- `➕ git add`: Prepara a intenção para ser registrada.
- `💾 git commit`: Colapso da função de onda. Registra o estado atual na história.
- `📥 git pull`: Sincroniza a realidade local com o fluxo externo.
- `📤 git push`: Projeta a tua realidade local para o Registro Akáshico remoto.
- `📦 git clone`: Materializa uma cópia de um universo distante localmente.
- `🌿 Branching`: Criação de realidades paralelas para experimentação.
- `🔀 Merging`: Fusão de linhas temporais em um único fluxo coerente.
- `🧾 git log`: Leitura da história das manifestações passadas.
- `🔄 git stash`: Reserva temporária de uma intenção incompleta no vácuo.
- `🔗 Remotes`: Conexões com as origens e fontes (origin/upstream).

### ⚔️ Gestão de Crise e Maestria

- `⚔️ Merge Conflicts`: Divergência de intenções que exige resolução manual.
- `🧠 Rebase`: Reescrevendo a história para uma linearidade mais limpa.
- `🔍 Bisect`: Busca binária na história para localizar a origem da decoerência (bug).
- `🧹 Cherry-pick`: Extração seletiva de uma manifestação específica de outra linha temporal.
- `🧱 Reset`: Retorno a um estado anterior, descartando manifestações indesejadas.
- `🪄 Revert`: Criação de uma nova manifestação que anula os efeitos de uma anterior.
- `🔍 Reflog`: O registro de todas as ações de versionamento, o Akasha do Git.

### 🧬 Estruturas Avançadas

- `🔒 Hooks`: Gatilhos automáticos disparados em eventos de versionamento.
- `🧬 Submodules`: Inclusão de universos independentes dentro do teu projeto.
- `🧬 Interactive Rebase`: A arte de lapidar a história antes da fusão final.

---

### ⚙️ ESTRATÉGIA DE REPOSITÓRIO

- **Mono vs Multi**: A decisão entre um único Grande Registro (Monorepo) ou múltiplos Registros Fragmentados (Polyrepo).
- **Debugging History**: O uso do registro histórico como ferramenta de cura do sistema.
